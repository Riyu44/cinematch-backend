2. Distributions

Rating distribution

Most movies cluster between 5–8. Ratings below 4 are rare after our 50-vote filter. We'll use rating as a feature in K-Means and as a re-ranking signal.

Popularity distribution (TMDB score)

Heavily right-skewed — a few blockbusters dominate. We'll apply log(popularity) in our collaborative filter to prevent blockbusters from drowning out niche films.

Runtime distribution (minutes)

Peak at 90–120 mins — the standard feature film. The user's "time commitment" filter maps directly onto these bins. Anything <30 or >240 will be capped/excluded.

Release year distribution

Dataset is recency-biased (expected — TMDB popularity favours newer films). Pre-2000 films are under-represented but present. Year won't be a model feature but can be a user filter.
3. Genre Distribution

Top genres by movie count (multi-label)

Drama and Action dominate — expected for a popularity-sorted dataset. Genre is our primary K-Means clustering feature and one of the user's input dimensions.
4. Why These 3 Models? — Design Rationale
🔵 Model 1 — KNN on Sentence Embeddings
Answers: "Give me movies semantically similar to the ones I loved."
Input features: title + overview + genres → 384-dim vector via all-MiniLM-L6-v2
Why this model: Overview text carries rich signal (tone, plot, theme) that genre labels alone miss. "Parasite" and "Knives Out" share no genre overlap but are semantically close.
Fill rate dependency: Needs overview ✓ and genres ✓
UnsupervisedContent-basedNo labels needed
🟢 Model 2 — K-Means Clustering
Answers: "Filter to movies in the right mood, genre and length."
Input features: genre (one-hot) + rating + runtime + popularity
Why this model: Groups movies into natural mood archetypes (dark thriller, feel-good comedy, epic action etc.) without any labels. User preference inputs map to clusters — pre-filtering the KNN search space from 1389 → ~70 movies.
Fill rate dependency: Needs genres ✓, runtime ✓, rating ✓
UnsupervisedDimensionality filter
🟣 Model 3 — Collaborative Filter (TruncatedSVD)
Answers: "People with your taste also loved these."
Input features: User-movie affinity matrix → 50 latent taste dimensions via matrix factorisation
Why this model: Captures latent taste patterns that content features miss. Someone who loves "The Godfather" + "Goodfellas" probably likes "Casino" — even though the overviews aren't similar enough for KNN to catch it.
Fill rate dependency: Needs rating ✓ + votes ✓ + popularity ✓
UnsupervisedLatent factorsMatrix factorisation
How the 3 models combine: K-Means narrows the candidate pool by mood/genre/runtime preference first. KNN and collaborative filter each score the remaining candidates independently. Final rank = KNN score × 50% + CF score × 30% + cluster proximity × 20%. This weighted ensemble handles the cold-start problem: new users with no history fall back to KNN + clustering until enough signal accumulates for CF.
